Technical Challenge: Integrated FastAPI, Streamlit, and OpenAI API, and engineered prompt design and parameter tuning to optimize ChatGPT translation output, achieving efficient and high-quality translations.

Future Vision: Plans for features like audio playback, progress visualization, and personalized review intervals to create an even more effective AI-powered language learning app.

🤖 Japanese Instruction-following LLM

東京大学 松尾研究室主催「大規模言語モデル講座」最終課題として開発。
日本語の大規模言語モデル「llm-jp-3-13b」をベースに、実際の質問に対する回答をより正確に返せるようにファインチューニングしました。
Unslothライブラリを用いて効率的に学習し、最終的には Hugging Face 上でモデルを公開しました。

Developed as the final project for the "Large Language Models Course" hosted by the Matsuo Laboratory at the University of Tokyo. This model is fine-tuned based on the Japanese large language model llm-jp-3-13b to provide more accurate responses to real-world questions. Using the Unsloth library, training was performed efficiently, and the resulting model has been published on Hugging Face.

技術的チャレンジ：

Unsloth・LoRAによる軽量ファインチューニングを行い、Google Colab上でモデルを学習。
4bit量子化や学習効率の工夫により、限られたGPUリソースでも安定した出力を得られるよう設計しました。
また、推論コードや出力形式（JSONL）の整備を行い、再現性のある形で成果を公開しました。

Technical Challenge: Efficiently fine-tuned a 13B Japanese LLM with Unsloth and LoRA using Google Colab. Used 4-bit quantization for memory savings and implemented reproducible inference with JSONL outputs and public model hosting.

将来の展望：

より精緻な指示理解や会話の一貫性を向上させる継続的なトレーニングや、日本語タスク用に最適化されたInstructionデータセットの自作を進めたい。
商用レベルの応答品質を目指した改良にも取り組みたい。

Future Vision: Further improvements in instruction-following quality, task-specific fine-tuning, and development of proprietary Japanese instruction datasets.

使用技術：

Python, Transformers, Hugging Face, Google Colab, Unsloth, LoRA, JSONL

📊 AirREGI アカウント獲得予測プロジェクト（GCI最終課題）

東京大学「グローバル消費インテリジェンス」寄附講座の最終課題として、AirREGIのアカウント獲得数を外部要因から予測するモデルを開発しました。

目的:

  • アカウント獲得数（acc_get_cnt）に影響する外部要因を定量的に分析
  • CM・祝日・通話件数などを考慮し、マーケティング施策に活かせる予測モデルを構築

使用技術・手法:

  • Python（pandas, seaborn, matplotlib, scikit-learn, XGBoost, SHAP）
  • 複数CSVの統合：calendar_data / cm_data / call_data / acc_get_data を日付で統合
  • XGBoostによる回帰モデル、SHAPで特徴量の重要度を可視化

分析結果:

  • 平日のCM放映がアカウント獲得数に大きく影響
  • 営業通話数（call_num）との相関は中程度
  • SHAPにより「CM」「曜日」「祝日」などが重要な特徴量として可視化され、モデルの透明性を確保

学びと工夫:

  • 時系列＋外部要因データの前処理・統合スキル
  • 予測モデルの解釈性向上
    - SHAP値による個別予測の要因分析
    - Feature Importanceによる全体的な特徴量寄与度評価
  • データ分析の結果をビジネス文脈に落とし込む視点
